**Descripción:**
Como sistema
Quiero almacenar los PDFs generados asociados a cada versión del documento y regenerarlos automáticamente cuando se publica una nueva versión
Para evitar regeneraciones innecesarias y garantizar que el PDF descargado siempre corresponde a la versión vigente del documento.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Almacenamiento en Object Storage):**
  **Dado que** el Job de generación PDF completa exitosamente,
  **Cuando** el PDF está listo,
  **Entonces** se almacena en Object Storage (S3 o compatible) con la clave `{document_id}/{version_number}/document.pdf` y el estado en `document_pdf_status` cambia a `ready`.

- [ ] **Escenario 2 (PDF válido mientras no haya nueva versión):**
  **Dado que** un PDF ha sido generado para la versión v1.0 del documento,
  **Cuando** el usuario solicita el PDF antes de que se publique una v2.0,
  **Entonces** el sistema devuelve el PDF cacheado de v1.0 sin regenerarlo.

- [ ] **Escenario 3 (Regeneración automática al publicar nueva versión):**
  **Dado que** se publica una versión v2.0 del documento,
  **Cuando** el nuevo snapshot es creado,
  **Entonces** se encola automáticamente un nuevo `GeneratePdfJob` para la v2.0; el PDF de v1.0 permanece disponible en el historial de versiones.

- [ ] **Escenario 4 (Consulta de estado del PDF):**
  **Dado que** el usuario quiere saber si el PDF de su documento está listo,
  **Cuando** consulta `GET /api/documents/{id}/pdf-status`,
  **Entonces** recibe el estado actual: `pending` (en cola), `processing` (Chromium trabajando), `ready` (URL disponible) o `failed` (con timestamp del error).

- [ ] **Escenario 5 (Acceso al historial de PDFs):**
  **Dado que** un documento tiene varias versiones publicadas con sus respectivos PDFs,
  **Cuando** el usuario consulta el historial de versiones,
  **Entonces** puede acceder al PDF de cada versión publicada de forma individual.

- [ ] **Requisito Técnico (Verificación de existencia en caché):**
  La verificación de si un PDF ya existe en el caché (HEAD request a S3) debe completarse en menos de 50 ms.

**MoSCoW:** `Must`

**Notas:**
- El bucket S3 es privado; ningún PDF es accesible de forma pública directa. Las descargas usan URLs firmadas temporales (F-08.4).
- La tabla `document_pdf_status` registra: `(document_version_id, status, s3_key, generated_at, error_message)`.
- Al publicar una nueva versión, el PDF anterior no se elimina; el historial de PDFs sigue al historial de versiones del documento (F-04.4).
