**Descripción:**
Como jefe de departamento
Quiero crear un borrador de programación didáctica y delegarlo a un profesor específico para que lo complete
Para gestionar la distribución de trabajo docente garantizando que el profesor asignado es el autor responsable a efectos de trazabilidad y Segregación de Funciones.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Delegación exitosa):**
  **Dado que** un usuario con permisos superiores ha creado un documento en Borrador,
  **Cuando** ejecuta la acción de delegar a un profesor específico,
  **Entonces** el documento aparece en la bandeja de borradores del profesor y desaparece de la del superior.

- [ ] **Escenario 2 (Profesor como autor para SoD):**
  **Dado que** un documento ha sido delegado a un profesor,
  **Cuando** el sistema evalúa las reglas de SoD,
  **Entonces** el `creator_id` del documento para efectos de Policies es el ID del profesor delegado, no el del superior que lo creó.

- [ ] **Escenario 3 (Superior no puede editar tras delegar):**
  **Dado que** un documento ha sido delegado,
  **Cuando** el superior intenta modificar el contenido de un bloque,
  **Entonces** el sistema devuelve HTTP 403; el superior perdió el acceso de edición al momento de la delegación.

- [ ] **Escenario 4 (Profesor no puede rechazar la delegación):**
  **Dado que** un documento ha sido delegado al profesor,
  **Cuando** el profesor accede a su bandeja de borradores,
  **Entonces** el documento aparece sin opción de "declinar" ni "rechazar"; no hay acción de rechazo disponible.

- [ ] **Escenario 5 (Registro en auditoría):**
  **Dado que** una delegación se realiza,
  **Cuando** la operación se confirma,
  **Entonces** se genera un registro en `audit_log` con el ID del delegante, el ID del profesor delegado, el ID del documento y el timestamp del servidor.

- [ ] **Requisito Técnico (Proceso de delegación):**
  La delegación debe completarse en menos de 200 ms incluyendo la actualización del `creator_id` y el registro en `audit_log`.

**Notas:**
- La delegación es definitiva: el superior no puede recuperar el acceso de edición a menos que el profesor le añada como colaborador (F-05.1).
- `DocumentController::delegate()` recibe un `DelegateDocumentDTO { document_id, assignee_id, delegated_by }` y delega en `DocumentService::delegate()`.
- El `DocumentService::delegate()` actualiza `creator_id` en el documento, registra en `audit_log` y dispara el evento de dominio para notificación externa.
- Solo usuarios con rol superior (Jefe de Departamento, Jefe de Estudios, Dirección) pueden ejecutar la delegación; la Policy verifica esto contra los datos del JWT.
