**Descripción:**
Como autor principal de una programación didáctica
Quiero compartir mi documento con otros usuarios asignando permisos de edición o solo lectura
Para permitir la co-autoría controlada sin perder el control sobre quién puede enviar el documento a revisión.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Añadir colaborador con permiso de edición):**
  **Dado que** el autor principal está en la vista de gestión de colaboradores,
  **Cuando** añade a un usuario con permiso `edit`,
  **Entonces** ese usuario puede abrir el documento y modificar el contenido de los bloques mientras el documento esté en estado Borrador.

- [ ] **Escenario 2 (Añadir colaborador con permiso de solo lectura):**
  **Dado que** el autor añade a un usuario con permiso `read`,
  **Cuando** ese usuario accede al documento,
  **Entonces** puede visualizar el contenido completo pero los controles de edición están deshabilitados y el botón de envío a revisión no aparece.

- [ ] **Escenario 3 (Solo el autor principal envía a revisión):**
  **Dado que** un colaborador con permiso `edit` ha realizado cambios en el documento,
  **Cuando** ese colaborador intenta enviar el documento a revisión,
  **Entonces** el sistema devuelve HTTP 403; únicamente el autor principal puede ejecutar esa acción.

- [ ] **Escenario 4 (Visibilidad del documento en listado del colaborador):**
  **Dado que** un documento ha sido compartido con un usuario,
  **Cuando** ese usuario accede a su listado de programaciones,
  **Entonces** el documento aparece con un indicador "Compartido contigo" que lo distingue de sus documentos propios.

- [ ] **Escenario 5 (Revocación de acceso):**
  **Dado que** el autor principal revoca el acceso de un colaborador,
  **Cuando** el colaborador intenta acceder al documento,
  **Entonces** el sistema devuelve HTTP 404 (el Global Scope filtra el documento como no accesible).

- [ ] **Requisito Técnico (Actualización de permisos):**
  La actualización de permisos de colaboración debe propagarse al Global Scope en menos de 200 ms; la próxima request del colaborador ya reflejará el cambio.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 4

**Notas:**
- Los colaboradores se registran en la tabla `document_collaborators` con `(document_id, user_id, permission)`.
- `DocumentService::addCollaborator(AddCollaboratorDTO $dto)` valida que el solicitante sea el autor principal.
- El Global Scope de `Document` se actualiza para incluir en los resultados los documentos donde el usuario es colaborador además de autor.
- Los co-autores con permiso `edit` son co-autores funcionales pero el autor principal es quien aparece como responsable en la auditoría y el SoD.
