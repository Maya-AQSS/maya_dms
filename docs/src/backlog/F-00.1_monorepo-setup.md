**Descripción:**
Como equipo de desarrollo
Quiero inicializar el monorepo Laravel 13 + React 19 con configuración de entornos (local, staging, producción), variables de entorno y estructura de directorios
Para que todos los desarrolladores puedan arrancar el proyecto con un único comando y los entornos sean reproducibles.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Estructura de directorios):**
  **Dado que** un desarrollador clona el repositorio,
  **Cuando** examina la estructura de carpetas raíz,
  **Entonces** encuentra dos directorios separados: `/backend` con la aplicación Laravel 13 y `/frontend` con la aplicación React 19, Vite y TypeScript.

- [ ] **Escenario 2 (Variables de entorno):**
  **Dado que** el repositorio está clonado,
  **Cuando** el desarrollador revisa el control de versiones,
  **Entonces** existe un archivo `.env.example` con todas las variables requeridas documentadas (DB, JWT secret, RabbitMQ, Redis, S3, Reverb) y el archivo `.env` real está excluido por `.gitignore`.

- [ ] **Escenario 3 (Comando de instalación):**
  **Dado que** el desarrollador tiene una máquina limpia con Docker instalado,
  **Cuando** ejecuta el comando `make install` o el script equivalente,
  **Entonces** el sistema instala las dependencias de ambos entornos (backend y frontend) sin intervención manual adicional.

- [ ] **Escenario 4 (Servicios de infraestructura local):**
  **Dado que** el entorno local está configurado,
  **Cuando** el desarrollador levanta los servicios,
  **Entonces** PostgreSQL, Redis y RabbitMQ están disponibles vía Docker Compose en los puertos configurados en `.env.example`.

- [ ] **Requisito Técnico (Tiempo de arranque):**
  El entorno local completo debe ser funcional en menos de 5 minutos en una máquina limpia con conexión a Internet estable.
