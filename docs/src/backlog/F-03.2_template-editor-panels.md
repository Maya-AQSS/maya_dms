**Descripción:**
Como creador de plantilla
Quiero editar el contenido y la estructura de una plantilla en un editor de 2 paneles (outline de bloques a la izquierda, propiedades y editor BlockNote a la derecha)
Para gestionar visualmente los bloques normativos sin necesidad de un wizard multi-paso.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Carga del editor):**
  **Dado que** un usuario accede a la edición de una plantilla,
  **Cuando** la pantalla carga,
  **Entonces** se muestra un layout de dos paneles: el panel izquierdo contiene el índice navegable de todos los bloques de la plantilla y el panel derecho muestra las propiedades del primer bloque junto al editor de contenido BlockNote.

- [ ] **Escenario 2 (Navegación entre bloques):**
  **Dado que** el editor está cargado,
  **Cuando** el usuario hace clic en un bloque del índice izquierdo,
  **Entonces** el panel derecho actualiza su contenido mostrando las propiedades y el contenido editable de ese bloque sin recargar la página.

- [ ] **Escenario 3 (Reordenación de bloques):**
  **Dado que** el usuario visualiza el outline de bloques,
  **Cuando** arrastra un bloque a otra posición en el índice,
  **Entonces** el orden de los bloques en la plantilla se actualiza inmediatamente y el cambio queda persistido al guardar.

- [ ] **Escenario 4 (Adición de bloques con comando /):**
  **Dado que** el cursor está activo en el editor BlockNote del panel derecho,
  **Cuando** el usuario escribe el comando `/`,
  **Entonces** aparece un menú contextual con los tipos de bloque disponibles (texto, tabla, imagen) y al seleccionar uno se inserta en la posición del cursor.

- [ ] **Escenario 5 (Bloques bloqueados visualmente diferenciados):**
  **Dado que** un bloque tiene estado `Bloqueado`,
  **Cuando** el usuario lo visualiza en el editor,
  **Entonces** el bloque aparece con un icono de candado y fondo diferenciado; el editor de contenido del panel derecho no es interactivo para ese bloque.

- [ ] **Escenario 6 (Autoguardado):**
  **Dado que** el usuario está editando contenido,
  **Cuando** transcurren 30 segundos sin guardar manualmente o el usuario navega a otro bloque,
  **Entonces** los cambios se guardan automáticamente y se muestra un indicador discreto de "Guardado" en la interfaz.

- [ ] **Requisito Técnico (Rendimiento de carga):**
  El editor debe estar completamente cargado e interactivo en menos de 2 segundos para plantillas con hasta 30 bloques.
