**Descripción:**
Como responsable de calidad ISO 9001
Quiero consultar el historial completo del ciclo de vida de cualquier documento o plantilla en una vista de línea de tiempo
Para responder a una inspección aportando la trazabilidad de quién realizó cada acción y en qué momento exacto.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Línea de tiempo del historial):**
  **Dado que** un documento ha pasado por múltiples estados y acciones,
  **Cuando** un usuario autorizado accede al panel de "Historial" del documento,
  **Entonces** ve una línea de tiempo cronológica con todos los eventos: creación, envíos a revisión, comentarios añadidos, aprobaciones, rechazos (con motivo), delegaciones y publicaciones.

- [ ] **Escenario 2 (Detalle por evento):**
  **Dado que** el historial está visible,
  **Cuando** el usuario revisa cada evento,
  **Entonces** cada ítem muestra: tipo de acción, nombre y ID del usuario que la realizó, timestamp exacto del servidor y descripción legible del cambio.

- [ ] **Escenario 3 (Historial de solo lectura):**
  **Dado que** un usuario accede al historial del documento,
  **Cuando** intenta modificar o eliminar cualquier entrada,
  **Entonces** no existe ningún control de edición en la interfaz; el historial es estrictamente de solo lectura.

- [ ] **Escenario 4 (Control de acceso al historial):**
  **Dado que** un usuario sin relación con el documento intenta acceder a su historial,
  **Cuando** hace la petición,
  **Entonces** el servidor devuelve HTTP 403; solo el autor, revisores asignados y usuarios de dirección pueden consultar el historial.

- [ ] **Escenario 5 (Exportación como JSON):**
  **Dado que** un auditor necesita integrar el historial con su sistema de gestión de calidad,
  **Cuando** solicita `GET /api/documents/{id}/history?format=json`,
  **Entonces** recibe el historial completo en formato JSON estructurado con todos los eventos y sus metadatos.

- [ ] **Requisito Técnico (Carga del historial):**
  El historial completo de un documento con hasta 100 eventos debe cargarse y renderizarse en menos de 500 ms.

**Notas:**
- La fuente de datos es la tabla `audit_log` (F-09.1); el historial se construye filtrando por `entity_id = :document_id` y ordenando por `timestamp ASC`.
- `AuditLogRepository::getDocumentHistory(string $documentId)` devuelve la colección de registros al `DocumentHistoryService`.
- El panel de historial en React 19 es un componente de línea de tiempo lateral accesible desde el menú del documento.
- La exportación JSON usa el mismo endpoint con el parámetro `?format=json`; el `DocumentHistoryResource` gestiona la serialización.
