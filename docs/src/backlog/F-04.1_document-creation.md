**Descripción:**
Como docente
Quiero crear una nueva programación didáctica con un solo clic desde la vista de un módulo específico
Para iniciar mi trabajo sin tener que seleccionar manualmente la plantilla correcta entre múltiples opciones.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Creación con plantilla única disponible):**
  **Dado que** el docente está en la vista de un módulo con exactamente una plantilla publicada disponible,
  **Cuando** pulsa el botón "Nueva Programación",
  **Entonces** el sistema crea el documento en estado Borrador vinculado a esa versión de plantilla y redirige al editor sin mostrar ningún formulario de selección.

- [ ] **Escenario 2 (Selector simple con múltiples plantillas):**
  **Dado que** el módulo tiene más de una plantilla publicada disponible,
  **Cuando** el docente pulsa "Nueva Programación",
  **Entonces** se muestra un selector simple con las opciones disponibles (nombre + descripción breve de cada plantilla); el docente selecciona una y el documento se crea sin formularios adicionales.

- [ ] **Escenario 3 (Creator_id desde JWT):**
  **Dado que** el documento se está creando,
  **Cuando** el sistema asigna el `creator_id`,
  **Entonces** el valor proviene exclusivamente del claim `sub` del JWT autenticado, nunca de un campo enviado por el frontend.

- [ ] **Escenario 4 (Aparición inmediata en el listado):**
  **Dado que** el documento acaba de crearse,
  **Cuando** el usuario vuelve al listado de programaciones,
  **Entonces** el nuevo documento aparece en la parte superior ordenado por fecha de creación.

- [ ] **Escenario 5 (Sin plantilla disponible):**
  **Dado que** el módulo no tiene ninguna plantilla publicada disponible para el docente,
  **Cuando** accede a la vista del módulo,
  **Entonces** el botón "Nueva Programación" no aparece o aparece deshabilitado con mensaje explicativo.

- [ ] **Requisito Técnico (Tiempo de creación y redirección):**
  Desde el clic en "Nueva Programación" hasta estar en el editor del documento nuevo deben transcurrir menos de 500 ms.

**Notas:**
- Principio Poka-yoke: reducir fricción al mínimo. Nunca mostrar un wizard de configuración para crear un documento.
- `DocumentController::store()` recibe un `CreateDocumentDTO` con `template_version_id`, `module_id` y `creator_id`; delega en `DocumentService::create()`.
- `DocumentRepository::findActiveTemplateForModule()` devuelve la versión publicada más reciente de la plantilla asignada al módulo.
- El `creator_id` lo establece el `DocumentService` desde el contexto de autenticación, nunca desde el DTO de entrada.
