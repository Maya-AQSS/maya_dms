**Descripción:**
Como sistema de gestión documental
Quiero versionar las plantillas mediante el patrón Snapshot en PostgreSQL
Para mantener un historial inmutable de cada versión publicada y permitir que los documentos existentes sigan funcionando con su versión original.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Creación de snapshot al publicar):**
  **Dado que** una plantilla en estado Borrador o En Revisión pasa a Publicado,
  **Cuando** se confirma la publicación,
  **Entonces** se crea un nuevo registro en `template_versions` con el JSONB completo de todos los bloques en ese momento, el número de versión correlativo y el UUID de cada bloque conservado.

- [ ] **Escenario 2 (Mensaje de changelog obligatorio):**
  **Dado que** un usuario intenta publicar una plantilla,
  **Cuando** el campo `changelog` está vacío en la petición,
  **Entonces** el servidor devuelve HTTP 422 y no crea el snapshot; el mensaje de error indica que el changelog es obligatorio.

- [ ] **Escenario 3 (Inmutabilidad del snapshot):**
  **Dado que** una versión de plantilla ha sido publicada y existe su snapshot en `template_versions`,
  **Cuando** cualquier usuario intenta modificar ese snapshot directamente,
  **Entonces** el sistema devuelve HTTP 403; el registro es de solo inserción (Append-Only).

- [ ] **Escenario 4 (Documentos vinculados a versión original):**
  **Dado que** un documento fue creado con la versión v1.0 de una plantilla y posteriormente se publica la v2.0,
  **Cuando** el usuario abre ese documento,
  **Entonces** el sistema carga los bloques de la versión v1.0 original; el documento no se migra automáticamente.

- [ ] **Escenario 5 (Historial de versiones consultable):**
  **Dado que** una plantilla tiene varias versiones publicadas,
  **Cuando** un usuario consulta el historial de versiones,
  **Entonces** recibe una lista cronológica con número de versión, fecha de publicación, autor y texto del changelog de cada versión.

- [ ] **Requisito Técnico (Rendimiento de consulta por versión):**
  La query `SELECT * FROM template_versions WHERE id = :version_id` debe responder en menos de 100 ms gracias al índice sobre `(template_id, version_number)`.

**Notas:**
- El versionado es estrictamente Append-Only: nunca se modifica ni elimina un snapshot publicado (Restricción 1, ADR-01).
- El UUID de cada bloque es inmutable entre versiones; esto permite anclar comentarios (F-06.2) y auditoría (F-09.1) al UUID del bloque.
- La lógica de creación del snapshot reside en `SnapshotService::createTemplateSnapshot(PublishDocumentDTO $dto)`.
- El `TemplateRepository` expone `findVersionById(string $versionId)` para consultas directas sin cargar la plantilla completa.
