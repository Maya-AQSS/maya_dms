**Descripción:**
Como docente
Quiero editar mi programación didáctica en un editor de 2 paneles con BlockNote
Para rellenar los bloques de contenido de forma intuitiva respetando las restricciones normativas definidas en la plantilla.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Layout de 2 paneles):**
  **Dado que** el docente abre un documento en edición,
  **Cuando** la pantalla carga,
  **Entonces** se muestra el panel izquierdo con el outline navegable de bloques (incluyendo indicadores de estado y badge obligatorio/opcional) y el panel derecho con las propiedades del primer bloque y el editor BlockNote.

- [ ] **Escenario 2 (Restricciones por estado de bloque):**
  **Dado que** un bloque tiene estado `Bloqueado`,
  **Cuando** el docente lo selecciona en el outline,
  **Entonces** el panel derecho muestra el contenido en modo lectura con icono de candado; el editor BlockNote no es interactivo para ese bloque.

- [ ] **Escenario 3 (Bloque Modificable — edición con registro):**
  **Dado que** un bloque tiene estado `Modificable`,
  **Cuando** el docente edita su contenido,
  **Entonces** el sistema registra el contenido original de la plantilla para poder generar el diff visual durante la revisión; el bloque NO puede eliminarse del documento.

- [ ] **Escenario 4 (Pegado de Markdown):**
  **Dado que** el docente tiene contenido en formato Markdown en el portapapeles,
  **Cuando** lo pega en el editor BlockNote,
  **Entonces** BlockNote interpreta el Markdown y lo convierte automáticamente al formato de bloques JSON estructurado, sin modo Markdown separado.

- [ ] **Escenario 5 (Adición de contenido con comando /):**
  **Dado que** el cursor está activo en un bloque Editable,
  **Cuando** el docente escribe el comando `/`,
  **Entonces** aparece el menú de tipos de contenido disponibles (texto, tabla, imagen) para insertar en ese bloque.

- [ ] **Escenario 6 (Guardado automático):**
  **Dado que** el docente ha realizado cambios en el contenido de un bloque,
  **Cuando** transcurren 30 segundos o el docente navega a otro bloque,
  **Entonces** los cambios se guardan automáticamente con indicador visual de "Guardado".

- [ ] **Requisito Técnico (Rendimiento de carga):**
  El editor debe ser completamente interactivo en menos de 2 segundos para documentos con hasta 50 bloques.

**Notas:**
- Reutilizar el mismo componente de 2 paneles de F-03.2 con las restricciones adicionales propias del editor de documentos.
- `DocumentController::updateBlock()` recibe un `UpdateBlockDTO { block_uuid, content, editor_action }` y delega en `DocumentService::updateBlock()`.
- El `DocumentService` verifica el estado del bloque (Bloqueado → 403, Modificable → guarda + registra original) antes de persistir via `DocumentRepository`.
- El guardado automático usa debouncing en el frontend: acumula cambios 30 s y envía una sola petición al backend.
