**Descripción:**
Como administrador del sistema
Quiero configurar la extensión FDW de PostgreSQL para acceder en modo solo lectura a la tabla de usuarios del dashboard corporativo
Para que Maya PD pueda consultar perfiles de usuario sin duplicar datos ni gestionar credenciales locales.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Extensión FDW habilitada):**
  **Dado que** la base de datos local de Maya PD está disponible,
  **Cuando** un administrador de base de datos verifica las extensiones instaladas,
  **Entonces** la extensión `postgres_fdw` aparece como activa en el catálogo de la base de datos.

- [ ] **Escenario 2 (Configuración de servidor y mapeo):**
  **Dado que** la extensión FDW está habilitada,
  **Cuando** se revisa la configuración de objetos de base de datos,
  **Entonces** existe un `FOREIGN SERVER` y un `USER MAPPING` configurados con las credenciales de solo lectura provenientes de variables de entorno (no hardcodeadas).

- [ ] **Escenario 3 (Tabla foránea de usuarios):**
  **Dado que** el servidor FDW está configurado,
  **Cuando** se consulta el esquema de la base de datos local,
  **Entonces** existe una `FOREIGN TABLE users_fdw` con las columnas mapeadas: `id`, `nombre`, `email` y `departamento`.

- [ ] **Escenario 4 (Permisos mínimos):**
  **Dado que** el usuario de aplicación Laravel intenta acceder a `users_fdw`,
  **Cuando** ejecuta una operación de escritura (INSERT, UPDATE o DELETE),
  **Entonces** la operación es rechazada; únicamente las consultas SELECT están permitidas para ese usuario.

- [ ] **Escenario 5 (Health check de conectividad):**
  **Dado que** el sistema arranca,
  **Cuando** se ejecuta el health check de infraestructura,
  **Entonces** el sistema verifica que la conexión FDW responde correctamente y reporta su estado.

- [ ] **Requisito Técnico (Rendimiento con caché):**
  Una consulta a `users_fdw` filtrada por `id` debe responder en menos de 200 ms sin caché. Con caché Redis activo, la misma consulta debe responder en menos de 5 ms.

**Notas:**
- Las credenciales FDW son distintas al usuario de aplicación de Laravel. Deben estar en variables de entorno separadas.
- En el entorno local de desarrollo, el dashboard corporativo se simula con un contenedor Docker con datos de prueba anónimos.
