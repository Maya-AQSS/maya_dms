---

**Descripción:**
Como inspector educativo
Quiero que el PDF oficial incluya metadatos de trazabilidad (fecha de generación, autor, versión del documento)
Para verificar la autenticidad del documento y cumplir con los requisitos de evidencia de auditoría de ISO 9001.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Metadatos visibles en la cabecera del PDF):**
  **Dado que** se genera el PDF de un documento publicado,
  **Cuando** el inspector abre el PDF,
  **Entonces** la cabecera del documento muestra: nombre del documento, nombre completo del autor, número de versión y fecha y hora de publicación (no de descarga).

- [ ] **Escenario 2 (Metadatos del snapshot — no del JWT activo):**
  **Dado que** el autor original cambió de nombre en el sistema externo después de publicar el documento,
  **Cuando** se genera o se descarga el PDF,
  **Entonces** el PDF muestra el nombre del autor tal como estaba registrado en el momento de publicación (datos del snapshot), no el nombre actual.

- [ ] **Escenario 3 (HTML generado en el backend):**
  **Dado que** el Job de generación PDF se procesa en el Worker,
  **Cuando** se prepara el contenido para Puppeteer,
  **Entonces** el HTML completo (incluyendo cabecera con metadatos y contenido de bloques) es generado íntegramente por el backend; no se acepta HTML del frontend como fuente para el PDF.

- [ ] **Escenario 4 (Sin QR ni hash en MVP):**
  **Dado que** el documento se publica y se genera el PDF,
  **Cuando** el inspector revisa el PDF,
  **Entonces** el PDF incluye los metadatos textuales (autor, versión, fecha) pero NO incluye código QR ni hash criptográfico; estos elementos están diferidos a post-MVP.

- [ ] **Requisito Técnico (Generación del HTML):**
  La generación del HTML con metadatos y contenido de bloques JSONB debe completarse en menos de 200 ms antes de enviarlo a Chromium.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 5

**Notas:**
- Los metadatos reflejan el estado en el momento de publicación (timestamp del servidor del snapshot), no el momento de descarga. Esto garantiza inmutabilidad incluso si el autor cambia de nombre posteriormente.
- La inyección de QR y hash criptográfico está explícitamente diferida a post-MVP (Hecho 10, Restricción 5).
- `PdfGenerationService::buildHtml(DocumentVersion $version)` transforma el JSONB del snapshot a HTML con la cabecera de metadatos.
