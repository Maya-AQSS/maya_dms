**Descripción:**
Como docente
Quiero ser informado de forma no intrusiva cuando existe una nueva versión de la plantilla en la que se basa mi programación
Para conocer los cambios normativos disponibles sin que el sistema modifique o bloquee mi documento actual.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Detección de nueva versión al abrir el editor):**
  **Dado que** un documento fue creado con la plantilla v1.0 y existe una v2.0 publicada,
  **Cuando** el docente abre el editor del documento,
  **Entonces** se muestra un banner informativo no intrusivo en la parte superior del editor indicando que existe una versión más reciente de la plantilla.

- [ ] **Escenario 2 (Sin banner cuando la versión es la más reciente):**
  **Dado que** un documento está basado en la versión más reciente publicada de su plantilla,
  **Cuando** el docente abre el editor,
  **Entonces** no aparece ningún banner y el editor se muestra sin elementos adicionales.

- [ ] **Escenario 3 (Enlace al changelog de la nueva versión):**
  **Dado que** el banner de nueva versión está visible,
  **Cuando** el docente hace clic en "Ver cambios",
  **Entonces** se abre un panel de solo lectura con el changelog de la nueva versión de la plantilla sin abandonar el editor.

- [ ] **Escenario 4 (El banner es descartable):**
  **Dado que** el banner está visible,
  **Cuando** el docente lo cierra manualmente,
  **Entonces** el banner desaparece en esa sesión; se vuelve a mostrar la próxima vez que el usuario abra el documento.

- [ ] **Escenario 5 (El documento no se altera):**
  **Dado que** el banner de nueva versión está visible,
  **Cuando** el docente continúa editando el documento,
  **Entonces** el documento sigue funcionando con la versión de plantilla original (v1.0); el sistema no migra ni bloquea el documento automáticamente.

- [ ] **Requisito Técnico (Comparación de versiones eficiente):**
  La comparación entre `template_version_id` del documento y la última versión publicada de la plantilla debe resolverse en menos de 50 ms (consulta de metadatos, sin cargar el JSONB completo de los bloques).

**Notas:**
- La migración side-by-side está explícitamente diferida a post-MVP (Hecho 14, Decisión D-26, D-39). Este banner es la solución MVP.
- El backend expone un endpoint `GET /api/documents/{id}/template-version-status` que devuelve `{ current_version, latest_version, has_update: bool, changelog }`.
- El frontend gestiona el estado de "banner descartado" en localStorage con clave `{document_id}_version_banner_dismissed`.
