**Secundaria:** `Integration`
**Impacta en:** Equipo Backend

**Prioridad:** `Must`

---

**Descripción:**
Como docente o inspector educativo
Quiero descargar el PDF oficial de una programación didáctica mediante un enlace seguro temporal
Para acceder al artefacto de auditoría sin exponer el almacenamiento privado de la organización a acceso público.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Descarga exitosa con PDF listo):**
  **Dado que** el usuario tiene acceso al documento y el PDF está en estado `ready`,
  **Cuando** hace la petición a `GET /api/documents/{id}/pdf`,
  **Entonces** el servidor genera una URL firmada de S3 con TTL de 15 minutos y responde con HTTP 302 redirigiendo al cliente a esa URL.

- [ ] **Escenario 2 (PDF en generación — respuesta 202):**
  **Dado que** el PDF está en estado `pending` o `processing`,
  **Cuando** el usuario solicita la descarga,
  **Entonces** el servidor devuelve HTTP 202 con la cabecera `Retry-After: 30` indicando que el PDF aún no está listo.

- [ ] **Escenario 3 (PDF fallido — respuesta 500 con opción de regenerar):**
  **Dado que** el Job de generación PDF falló tras los 3 reintentos,
  **Cuando** el usuario solicita la descarga,
  **Entonces** el servidor devuelve HTTP 500 con mensaje descriptivo del error y un endpoint sugerido para solicitar la regeneración manual.

- [ ] **Escenario 4 (Verificación de permiso antes de generar URL):**
  **Dado que** un usuario sin acceso al documento solicita el PDF directamente con el `document_id` de otro usuario,
  **Cuando** el endpoint procesa la petición,
  **Entonces** el Global Scope impide el acceso y el servidor devuelve HTTP 404 (sin revelar la existencia del documento).

- [ ] **Escenario 5 (Bucket S3 privado — sin acceso público):**
  **Dado que** el PDF está almacenado en S3,
  **Cuando** se intenta acceder directamente a la URL de S3 sin firma,
  **Entonces** S3 devuelve HTTP 403; solo las URLs firmadas con TTL válido permiten la descarga.

- [ ] **Requisito Técnico (Tiempo de generación de URL firmada):**
  El tiempo desde la petición hasta la redirección 302 (verificación de permisos + generación de URL firmada) debe ser inferior a 200 ms.

---

**MoSCoW:** `Must`
**Sprint:** Sprint 5

**Notas:**
- La URL firmada expira en 15 minutos. Si el usuario necesita volver a descargar, debe hacer una nueva petición al endpoint.
- El bucket S3 tiene política de acceso privado; nunca se configura como público.
- `PdfDownloadService::getSignedUrl(string $documentId, string $requesterId)` verifica permiso via `DocumentRepository` (con Global Scope) y genera la URL firmada via el SDK de S3.
