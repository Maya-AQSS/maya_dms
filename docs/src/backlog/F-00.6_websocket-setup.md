**Descripción:**
Como equipo de desarrollo
Quiero configurar el servidor WebSocket con Laravel Reverb y Echo para habilitar canales de presencia y locks de bloques en tiempo real
Para que la edición colaborativa pueda implementarse sobre una base de comunicación bidireccional segura.

**Criterios de Aceptación (Checklist):**

- [ ] **Escenario 1 (Laravel Reverb configurado):**
  **Dado que** el entorno está levantado,
  **Cuando** se revisa la configuración de broadcasting,
  **Entonces** Laravel Reverb está instalado y configurado como driver de broadcasting activo.

- [ ] **Escenario 2 (Autenticación de canales):**
  **Dado que** un cliente React intenta unirse a un canal privado o de presencia,
  **Cuando** el servidor recibe la petición de autorización,
  **Entonces** el endpoint `/broadcasting/auth` verifica el JWT del usuario y comprueba que tiene permisos de acceso al documento antes de autorizar el canal.

- [ ] **Escenario 3 (Cliente Echo configurado):**
  **Dado que** el frontend React 19 está cargado,
  **Cuando** se pierde la conexión WebSocket,
  **Entonces** el cliente Echo intenta reconectarse automáticamente sin intervención del usuario.

- [ ] **Escenario 4 (Heartbeat y timeout):**
  **Dado que** un cliente está conectado al servidor WebSocket,
  **Cuando** pasan 30 segundos sin actividad,
  **Entonces** el cliente envía un ping al servidor; si no recibe respuesta en 60 segundos, el servidor considera la conexión caída y libera los recursos asociados.

- [ ] **Escenario 5 (Configuración de credenciales):**
  **Dado que** se configura el servidor WebSocket,
  **Cuando** se revisa la variable de entorno correspondiente,
  **Entonces** la variable `REVERB_APP_KEY` es independiente y diferente a la clave JWT del dashboard corporativo.

- [ ] **Requisito Técnico (Latencia de mensajes):**
  La latencia de entrega de un mensaje WebSocket debe ser inferior a 100 ms en una red local.
