# Registro de Cambios y Nuevos Requisitos

> **Cómo usar este archivo:**
>
> 1. Añade cada cambio o nuevo requisito como un bloque `CHANGE-XX` con numeración correlativa.
> 2. Consulta la plantilla en `docs/templates/change-request.md` para el formato completo.
> 3. Cuando hayas registrado todos los cambios de una sesión, ejecuta `/revisar-cambios`.
> 4. El agente leerá este archivo, analizará el impacto sobre todos los backlogs existentes,
>    te mostrará una **matriz de impacto** para tu aprobación y luego aplicará los cambios.
>
> **Importante:** No borres los cambios ya procesados. Quedan como historial de evolución del proyecto.
> Cada CHANGE-XX procesado quedará registrado también en `AUDIT_LOG.md`.
